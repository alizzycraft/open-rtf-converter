# Fuzz Targets

These targets are for `cargo fuzz` and intentionally live outside the main crate. The fuzz crate depends on `open-rtf-converter` with `default-features = false` so the harness exercises the memory-only core without pulling in the CLI wrapper.

Useful runs:

```powershell
cargo fuzz run rtf_lexer
cargo fuzz run rtf_parser
cargo fuzz run rtf_to_pdf_passive
```

The `rtf_to_pdf_passive` target treats every successful conversion as a security assertion: the generated bytes must pass the passive PDF audit.
